# タスク管理 — 詳細ガイド

`/task` と GitHub Issues / Projects v2 を組み合わせたタスク管理機能の使い方マニュアル。

---

## 1. 全体の構造

| 要素 | 役割 | 実体 |
|---|---|---|
| **Issue** | タスク1件の詳細。タイトル・本文・コメント・状態（open/closed）を持つ。 | GitHub Issue |
| **Projects v2 ボード** | 全タスクの俯瞰ビュー。1リポジトリに1つのボードに、すべての Issue が並ぶ。 | Projects v2 |
| **Status** | Todo / In Progress / Done | Projects 単一選択フィールド |
| **案件** | タスクを業務テーマで分類（例: 「新チャンネル」「動画リサーチ」） | Projects テキストフィールド |
| **期日** | 締切日（YYYY-MM-DD） | Projects 日付フィールド |
| **優先度** | 高 / 中 / 低 | Projects 単一選択フィールド |
| **工数(時間)** / **工数(日)** | 見積もり工数。タスクごとに片方/両方を使い分け可。 | Projects 数値フィールド × 2 |

**設計方針**: 1リポジトリ = 1Projects ボード。案件ごとに別ボードを作るのではなく、ボード内の「案件」フィールドで分類する。

> 注: GitHub の「Sub-issues」機能（親子関係）は **現状未対応**。タスクはフラットに並ぶ。

---

## 2. タスクの作成

### スラッシュコマンド（推奨）
自然文で書くと Claude が解釈して各フィールドに振り分ける:

```
/task コテンの図解#2を取り込む 案件:オスマン特集 期日:5/30 優先度:高 工数:3時間
/task 動画台本を書く 案件:新チャンネル 期日:6/2 優先度:中 工数:1日
/task SNS投稿の予約をまとめる 案件:プロモ 工数:30分
```

「工数:30分」「工数:0.5日」のような自然な表現も Claude が正しい単位に変換する。
迷う表現（「工数:5」など単位省略）は確認される。

### CLI 直接
```
brain task add "タイトル" \
  --project "案件名" \
  --due 2026-05-30 \
  --priority 高 \
  --effort-hours 3 \
  --effort-days 1 \
  --note ZK-20260523-001234 \
  --body "詳細メモ"
```

`--note` を指定すると、関連 ZK ノートの `tasks:` フィールドに Issue 番号が逆リンクされる（双方向）。

### 作成時に起きること
1. GitHub Issues に1件作成（`#N` 番号付き、URL あり）
2. Projects ボードに自動追加、Status=Todo
3. 指定したフィールドに値が入る
4. `--note` 指定時はノート側にも逆リンクが付き、commit+push される

## 3. タスクの確認

### CLI
```
brain task list                              # 全タスク
brain task list --status "In Progress"       # 作業中のみ
brain task list --project "新チャンネル"     # 案件で絞り込み
brain task show 12                           # #12 の詳細（フィールド値・本文）
```

### GitHub Web UI
- リポジトリ → **Issues** タブ: リスト形式の一覧（GitHub のデフォルト）
- リポジトリ → **Projects** タブ → 該当ボード: カンバン / テーブル / ロードマップ

## 4. Projects v2 ボードを使いこなす（カンバン・案件別ビュー）

ボードは複数の **ビュー** を保存できる。同じデータを、用途別に違う見せ方で使う。
GitHub UI 上で1回設定すれば、次回からタブで切り替えるだけ。

### おすすめの初期ビュー構成

| ビュー名 | 設定 | 用途 |
|---|---|---|
| **カンバン** | Layout: Board / Group by: Status | Todo・In Progress・Done の3列カンバン。進捗の俯瞰。 |
| **案件別** | Layout: Board or Table / Group by: 案件 | 案件ごとにタスクが固まる。案件単位の作業計画。 |
| **期日順** | Layout: Table / Sort: 期日 ↑ | 締切が近い順。「今週やること」の確認。 |
| **優先度: 高** | Filter: 優先度 = 高 | 火急のタスクのみ表示。 |
| **未割り当て** | Filter: 案件 is empty | 案件未指定のタスクを拾い上げる。 |

### ビューの作り方（GitHub UI 操作）

1. Projects ボードを開く
2. ビュー切替バー右端の **`+ New view`** をクリック
3. 名前を入力（例: 「案件別」）
4. `Layout` / `Group by` / `Filter` / `Sort` を設定
5. 自動保存。次回からビューを切り替えるだけで再現できる

### グループ化の挙動
- `Group by 案件` → 案件ごとに列やセクションが分かれる
- 列ヘッダで折りたたみ/展開可能。タスク数が増えても俯瞰しやすい
- グループ間のドラッグ&ドロップで案件変更も可能（UI 上で）

## 5. タスクの更新・完了

### 状態を変える
```
brain task update 12 "In Progress"     # 作業開始
brain task update 12 Done              # 完了（Issue もクローズ）
brain task done 12                     # 完了の別名
brain task update 12 Todo              # 再オープン
```

スラッシュコマンドでもOK: `/task #12 を作業中に` / `/task #12 完了` / `/task #12 を Todo に戻す`

### フィールドを更新
```
brain task update 12 --priority 高
brain task update 12 --due 2026-06-15
brain task update 12 --effort-hours 5
brain task update 12 --effort-days 0.5
brain task update 12 --project "別の案件"
```

状態とフィールドを同時に更新するときは並べる:
```
brain task update 12 "In Progress" --due 2026-06-10 --priority 高
```

## 6. メモを追加する（タスクへの追記）

**現状、CLI からタスク本文／コメントに追記するコマンドは未実装。** 次のいずれかで対応する:

- **GitHub UI（推奨）**: Issue ページを開いて本文を編集、もしくはコメント追加
- 将来的に `brain task comment <番号> "メモ"` を追加することは可能（希望があれば対応）

タスクの **本文** は Issue 作成時に `--body` で初期投入できる。コメント機能は GitHub UI から。

## 7. ノートとの連動

### `/planning` → タスク化
1. `/planning` で企画ノートを作って会話
2. 会話の中で「やること」が出てきたら、私が「これをタスクにしますか？」と確認
3. OK なら自動で `brain task add --note <企画ノートID>` 実行
4. 企画ノートの `tasks:` フィールドに Issue 番号が逆リンクされる
5. タスクの本文にも `**関連ノート:** ZK-XXXX` が入る

### `/memo` → タスク化
- `/memo` の会話中、明らかな「やること」が出たら同様の流れで提案される

### 双方向の辿り方
- **タスク → 関連ノート**: Issue 本文の「関連ノート: ZK-…」を見る
- **ノート → 関連タスク**: ノートの frontmatter `tasks: [12, 14]` を見る

## 8. 典型的なワークフロー例

### 例 A: 案件「新チャンネル立ち上げ」のタスクを並べる

```
/task コンセプト確定 案件:新チャンネル 期日:5/30 優先度:高 工数:1日
/task SNS設計 案件:新チャンネル 期日:6/1 優先度:中 工数:5時間
/task 機材リストアップ 案件:新チャンネル 期日:6/3 優先度:低 工数:2時間
```

進捗確認:
```
brain task list --project "新チャンネル"
```

Projects ボードを「案件別」ビューで開けば、**新チャンネル** 列に3件が並ぶ。

### 例 B: 企画ノートから派生

1. `/planning 切り抜き運用の改善` → ブレスト会話 → 企画ノート保存
2. 会話の中で「ガイドライン作成・運用者教育・ツール選定」というタスクが出る
3. 私が「3件タスク化しますか？」→ OK
4. 3件が `案件: 切り抜き運用` 付きで作成され、企画ノートに逆リンク

### 例 C: 朝の確認

```
brain task list --status "In Progress"
brain task list --status Todo | head -10
```

または Projects ボードの「期日順」ビューを開いて、今週/今日のタスクを確認。

## 9. 既知の制限・今後の選択肢

| 項目 | 現状 | 拡張するなら |
|---|---|---|
| **案件 = Projects?** | 案件はテキストフィールド。1リポ1ボードで全案件が同居。 | 案件ごとに別 Project を持つ「マルチプロジェクト」化が可能。 |
| **小タスク（サブ Issue）** | Issue の親子関係は未実装。タスクはフラット。 | GitHub の sub-issues 機能で実装可（`--parent <番号>`）。 |
| **CLI からのメモ追記** | 未実装。GitHub UI 直接編集 or コメント機能で代用。 | `brain task comment <番号> "メモ"` を追加可能。 |
| **コメント表示** | `brain task show` はフィールド値と本文のみ表示（コメント未表示）。 | コメントも合わせて表示するよう拡張可能。 |

これらは現状の制限であって、必要なら追加実装する。希望を伝えてもらえれば対応する。
